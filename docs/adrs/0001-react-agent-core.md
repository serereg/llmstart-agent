# ADR 0001 — Agent Core на LangChain ReAct + tools

> **Статус:** ✅ Accepted
> **Дата:** 2026-06-21
> **Автор:** LLMStart Agent team
> **Область:** продукт LLMStart Agent
> **Supersedes:** —

---

## 1. Контекст

### 1.1 Текущая ситуация

LLMStart Agent — мультиканальный ассистент с воронкой: консультация → подбор продукта → оплата → лид. Нужно ядро, которое вызывает внешние действия (RAG, каталог, платежи, CRM).

### 1.2 Что породило необходимость решения

Агент должен не только генерировать текст, но и вызывать инструменты по ходу диалога. Курс фокусируется на архитектуре агента — нужен проверенный паттерн.

### 1.3 Силы, действующие на решение

| Сила | Вес | Следствие |
|------|-----|-----------|
| Учебная ценность | Высокий | ReAct — стандартный паттерн курса |
| Production-grade | Высокий | LangChain — зрелая экосистема tools |
| Простота MVP | Средний | Один Core, без микросервисов |

---

## 2. Рассмотренные варианты

### Вариант A. LangChain ReAct + tools

Суть: FastAPI-сервис с ReAct-агентом и набором tools (`search_knowledge_base`, `list_b2c_products`, и др.).

**Плюсы:**
- Прозрачные шаги для Langfuse и UI (reasoning, tools)
- Соответствует программе курса

**Минусы:**
- Зависимость от LangChain API

**Вердикт:** принят

### Вариант B. Прямые вызовы OpenAI function calling без оркестратора

**Плюсы:** меньше абстракций

**Минусы:** больше ручного кода для цикла agent loop

**Вердикт:** отклонён — дублирование логики ReAct

---

## 3. Решение

**Принимаем: вариант A — LangChain ReAct + tools в backend (FastAPI).**

### 3.1 Принципы

- Вся agent loop логика — в `backend/`
- Каналы не вызывают tools напрямую

### 3.2 Детали реализации

- Tools: `search_knowledge_base`, `list_b2c_products`, `save_lead`, `create_payment_link`, `confirm_payment`
- LLM: OpenAI API
- Traces: Langfuse callback

---

## 4. Последствия

### 4.1 Позитивные

- Единая точка для бизнес-логики агента
- Observability из коробки через Langfuse

### 4.2 Негативные (и митигация)

| Риск | Митигация |
|------|----------|
| Смена API LangChain | Фиксация версий в uv.lock |
